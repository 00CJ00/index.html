En esta sección, exploraremos los atributos en HTML, que son
propiedades adicionales que se pueden agregar a las etiquetas para proporcionar información adicional sobre un elemento.
Los atributos se definen dentro de la etiqueta de
apertura y generalmente vienen en pares de nombre y valor.

Los atributos globales son propiedades que se pueden aplicar a cualquier
elemento HTML para modificar su comportamiento o estilo, incluso si el elemento no está especificado en el estándar. Algunos de
los más comunes son id, class, style y title. Aunque se pueden aplicar a todos los elementos, su efecto varía y algunos no tendrán
un impacto visible en ciertos elementos, como hidden en un elemento .

Características clave

Aplicabilidad universal: Se pueden usar en todos los elementos HTML, incluidos los del . Funcionalidad: Permiten añadir parámetros adicionales que
modifican el diseño o la funcionalidad de un elemento.
Sintaxis: Tienen la misma sintaxis que otros atributos (nombre="valor"),
aunque algunos son binarios y no requieren un valor para activarse (por ejemplo, hidden).

Ejemplos comunes

id: Asigna un identificador único a un elemento, útil para estilos CSS y manipulación con JavaScript.
class: Permite agrupar elementos bajo un mismo nombre para aplicar estilos o realizar acciones colectivas.
style: Permite aplicar estilos CSS directamente a un elemento.
title: Proporciona información adicional sobre el elemento, que se muestra como una descripción emergente cuando el usuario lo pasa por encima.
lang: Indica el idioma del contenido del elemento.
data-*: Permite almacenar datos personalizados para usar con JavaScript u otros scripts.

2.- atributos form

La etiqueta
en HTML incluye varios atributos clave para controlar el comportamiento y el envío de datos del formulario.
El atributo action especifica la URL o el archivo donde se enviarán los datos del formulario cuando
se envíe, pudiendo ser una ruta de servidor o incluso un correo electrónico
con el prefijo mailto:. El atributo method define el método
HTTP utilizado para enviar los datos, siendo get para incluir los datos en la URL y post para enviarlos
en el cuerpo de la solicitud, siendo este último preferido por el W3C por razones de privacidad.

El atributo name proporciona un identificador único para el formulario,
lo que facilita su acceso mediante scripts del lado del cliente o procesamiento del lado del servidor.
El atributo target indica dónde se abrirá la respuesta del formulario, por ejemplo, _blank para abrir en una nueva pestaña.

Para formularios que incluyen archivos, el atributo enctype especifica
la codificación de los datos, siendo crucial para el envío de archivos.
El atributo autocomplete activa o desactiva el autocompletado de
los campos del formulario, con el valor por defecto siendo on.

Además, el atributo novalidate desactiva la validación HTML5 del formulario,
lo que puede ser útil cuando se desea manejar la validación mediante JavaScript.
Otro atributo relevante es accept-charset, que fuerza el uso de una codificación específica para los parámetros de texto del formulario.

Atributos input

Estos atributos permiten una gestión flexible y segura del envío de datos en formularios web.


La etiqueta HTML input> permite crear controles interactivos para formularios web, y cuenta con una amplia variedad de atributos que definen su comportamiento, apariencia y funcionalidad. Algunos de los atributos más comunes incluyen:
name: Asigna un nombre al campo, útil para identificarlo al enviar el formulario y procesar los datos en el servidor.

type: Define el tipo de entrada esperada, como texto (text), búsqueda (search), correo electrónico (email), URL (url), número telefónico (tel), contraseña (password), número (number), rango (range), fecha (date), entre otros.

value: Establece el valor predeterminado del campo, que se envía al servidor cuando se envía el formulario.

placeholder: Muestra un texto de sugerencia dentro del campo cuando está vacío; este texto desaparece cuando el usuario comienza a escribir.

required: Hace que el campo sea obligatorio, mostrando un mensaje de error si el usuario intenta enviar el formulario sin completarlo.

maxlength: Limita el número máximo de caracteres que se pueden ingresar en el campo.

min y max: Establecen los valores mínimo y máximo permitidos para campos numéricos o de fecha.

step: Define el incremento o decremento para campos numéricos o de rango.

autocomplete: Controla si el navegador puede autocompletar el valor del campo; puede tener valores como

on, off, o valores semánticos como name, email, street-address.

autofocus: Establece el foco en el campo al cargar la página, permitiendo al usuario comenzar a escribir inmediatamente.

disabled: Desactiva el campo, impidiendo que el usuario interactúe con él.

readonly: Hace que el campo sea de solo lectura, permitiendo ver el valor pero no modificarlo.

size: Define el ancho visual del campo, aproximadamente en número de caracteres.

list: Asocia el campo con una lista de opciones predefinidas, útil para sugerencias.

form: Permite asociar el campo con un formulario específico, incluso si está fuera del elemento form>.

accept: Solo aplicable cuando type="file", especifica los tipos de archivos que el servidor acepta.

pattern: Define una expresión regular que el valor del campo debe cumplir para ser válido.

Estos atributos, junto con los tipos de
entrada disponibles en HTML5, permiten crear formularios más semánticos, accesibles y con validación integrada,
mejorando la experiencia del usuario y reduciendo la necesidad de código JavaScript para validaciones básicas.
